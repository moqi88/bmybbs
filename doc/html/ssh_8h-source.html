00296 /* Socket for connecting the authentication agent.  Normally the connection 
00297    to the authentication agent is passed in a file descriptor; however,
00298    on some systems, commonly used shells close all open file descriptors.
00299    To make the agent usable on those systems, configure checks whether
00300    the shells close all descriptors, and if so, uses a socket instead.
00301    That socket is an unix-domain socket and must not be accessible by
00302    anyone but the user him/herself. A directory \'ssh-agent-<loginname>\'
00303    is created under /tmp, which is supposedly on the local machine,
00304    and socket is created under the directory. On some systems, sockets\'
00305    protections are not adequately checked, so this mode 700 per-user

00307    the daemon (running as root) might not be able to create and listen
00308    to the socket.
00309

00311    for example), but only the last directory in the path will be
00312    dynamically created and deleted by sshd and ssh-agent. */
